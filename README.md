# resource-portal-web

## 项目简介

`resource-portal-web` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，旨在提供一个干净、通用的页面存放与分发空间。每个 HTML 页面均为独立资源，可被直接访问或嵌入其他环境使用。

## 目录说明

```
resource-portal-web/
├── pages/          # 存放所有独立 HTML 页面
├── assets/         # 公共资源文件（CSS、JavaScript、图片等）
├── index.html      # 仓库入口页面（可选）
└── README.md       # 本文件
```

- **pages/**：该目录下包含各个独立 HTML 页面，每个页面以有意义的文件名命名，便于识别和引用。
- **assets/**：存放页面间共享的静态资源，如样式表、脚本文件、图标或字体。
- **index.html**：可选的起始页面，用于展示仓库内容或提供导航。

## 页面归档说明

所有 HTML 页面均以纯静态方式存放，无需后端服务即可直接通过浏览器打开。每个页面独立完整，不依赖外部不可控资源。归档时建议遵循以下约定：

- 页面文件名使用英文小写、连字符分隔（如 `example-page.html`）。
- 每个页面应在 `<head>` 中添加简要的 `<meta>` 描述。
- 避免在页面中嵌入敏感信息或动态依赖。

## 维护说明

### 添加新页面

1. 将 HTML 文件放入 `pages/` 目录。
2. 如有需要，将相关资源放入 `assets/` 目录，并在 HTML 中通过相对路径引用。
3. 提交 Pull Request 或直接推送到主分支。

### 更新现有页面

直接修改对应 HTML 文件即可。修改后请确保页面功能正常，并更新相关描述（如有）。

### 删除页面

如需删除某个页面，请同时移除其在 `pages/` 中的文件，并检查是否有其他页面引用该资源。

## 注意事项

- 本仓库仅用于存放和发布静态 HTML 内容，不包含任何动态后端逻辑。
- 请勿上传包含恶意代码、侵犯版权或违反法律法规的内容。
- 所有页面应保持通用性，避免硬编码特定域名或服务。
